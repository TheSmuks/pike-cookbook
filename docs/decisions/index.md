## Using ADRs

ADRs document the context, decision, and consequences of significant choices. They help:

- Onboard new contributors
- Understand why decisions were made
- Track evolution of the architecture
- Avoid repeating past mistakes

## Submitting an ADR

1. Create a new file: `docs/decisions/NNN-title.md`
2. Follow the [ADR template](template.md)
3. Submit a PR for review
4. Update this index after acceptance
